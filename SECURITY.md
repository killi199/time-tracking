# Security Policy

We take the security of `time-tracking` seriously and are committed to keeping our users safe. Because this is an open-source project maintained by volunteers, we appreciate your help in responsibly disclosing any issues you find.

## Supported Versions

We provide security updates only for the **latest stable version** of `time-tracking`.

We do not provide security patches for older versions, beta releases, or forks of the project. Please ensure you are testing against the latest release before submitting a vulnerability report.

## Reporting a Vulnerability

If you discover a security vulnerability, please disclose it to us responsibly.

**DO NOT OPEN A PUBLIC ISSUE FOR SECURITY VULNERABILITIES!**

Publicly disclosing a vulnerability gives attackers an opportunity to exploit it before we can release a patch. Instead, please report vulnerabilities privately using **GitHub's Security Advisories**:

1. Navigate to the **Security** tab of this repository.
2. Click on **Report a vulnerability** to privately disclose the issue.
3. Provide a clear description, the realistic impact, and detailed steps to reproduce.

We will do our best to review your report promptly. We ask that you give us up to **90 days** to investigate and release a fix before making any information public.

## What We Do Not Accept

We generally do not accept reports for:

- Theoretical attacks without proof of exploitability.
- Issues originating from third-party libraries (please report these to the library maintainers).
- Automated scanner reports without context.

## Recognition

While we cannot offer bug bounties for security reports, we are more than happy to give you credit for the discovery in our release notes and Security Advisories.
